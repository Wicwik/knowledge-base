# resources/queue/

Things you plan to read, watch, or listen to but haven't yet.

**What belongs here:** One Markdown file per resource, using the standard frontmatter (title, type, author, year, format, status, url). Set `status: queued` when you add it; update to `in-progress` or `done` as you go.

**Who fills it:** You (or Claude when you ask it to queue something).

**Example:**

```
resources/queue/editing-model-task-arithmetic.md
---
title: Editing Models with Task Arithmetic
type: paper
author: Ilharco et al.
year: 2023
format: url
status: queued
url: https://arxiv.org/abs/2212.04089
---
```

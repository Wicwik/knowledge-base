# daily/

One Markdown file per day for fleeting notes — things you want to capture quickly without deciding where they belong yet.

**What belongs here:** Raw thoughts, meeting notes, reading reactions, half-formed ideas. One file per day, named `YYYY-MM-DD.md`.

**Who fills it:** You.

**Example:**

```
daily/2026-04-07.md
---
Read the iso-merging paper. The isotropic scaling idea is surprisingly simple.
Check whether SAR metric applies to LoRA merging too.
---
```
